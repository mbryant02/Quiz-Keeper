📚 Quiz Keeper

Complete user guide — building questions, generating tests, and working with exports.

Overview

Thanks for trying Quiz Keeper, I made this tool at the request of my amazingly hardworking wife who works as a university professor and found the tools provided by the university system and the book publisher's online platform to be frustrating. I put a lot of thought into the implementation of this tool and also of course had my wife's suggestions and feedback, so hopefully you will find it usefull too. This tool is provided free and open source under the GNU GPLv3 License, and while I have done my best to make sure it works correctly, a robust amount of testing has not yet been performed. Use it at your own risk and always have a backup of your database saved. In that same vein, if you encounter bugs or have suggestions I cannot promise a response or action but feel free to contact me.

Quiz Keeper is a browser-based tool that lets you build and maintain a reusable question bank, then quickly compose, preview, and export professional tests as PDF or Word (DOCX) documents. All data is stored locally in your browser using IndexedDB — no internet connection should be required.

ℹ️
Local Storage

Because all data lives in your browser's site data, clearing browser data or switching to another device permanently can lose your questions data file.

If you switch browsers, or use a different machine your question bank will not be available unless you have the database file with you for import, either backed up on a drive or in cloud, but it should be safe where you originally started it. See Database Backup below.

⚠️ Database Backup — Read This First

🚨
Your data can be lost permanently without a backup.

The following actions will or may irreversibly erase your entire question bank and test history:

  • Clicking ⚙️ Database → 🗑️ Clear Database
  • Clearing your browser's site data or cache
  • Uninstalling or resetting your browser
  • Using a different browser or computer
  • Browser storage eviction (low disk space)

There is no undo. Always keep a current backup file on an external drive or cloud storage.

How to Back Up

  1. Click ⚙️ Database in the top-right header.
  2. Click 💾 Export Database.
  3. A .json file is downloaded automatically. Save it somewhere safe — a cloud folder, external drive, or email it to yourself.
💡
Recommended backup schedule

Export the database after every significant session of adding or editing questions. The backup database files are prenamed with the export date, but if you wish to rename them it is recommended to include the date (e.g., questionbank_2026-03-02.json) so you can roll back to an earlier version if needed.

How to Restore from a Backup

  1. Click ⚙️ Database → 📂 Import Database.
  2. Select your previously exported .json backup file.
  3. Click Import. The data will be merged into the current database.
⚠️
Merges, not replaces

Importing adds data on top of what already exists. If you import the same file twice, duplicate questions will appear. To restore to a clean state, clear the database first, then import.

Collections

Collections (Subjects, Classes, and Exam Periods) are the organisational backbone of the question bank. Set these up before adding questions so that every question can be tagged and filtered correctly.

Go to the Collections tab to manage them.

Subjects

A Subject is the top-level category (e.g., Biology, Mathematics, World History). Every Class and Exam Period belongs to a Subject.

Click + New Subject, enter a name and optional description, then click Save Subject.

Classes

A Class belongs to a Subject and represents a specific group of students (e.g., Grade 10A, Advanced Placement). Classes allow you to scope questions and tests to particular groups.

  1. Click + New Class.
  2. Select the parent Subject.
  3. Enter a Class Name and optional description.
  4. Click Save Class.

Exam Periods

Exam Periods belong to a Subject and a Class (e.g., Midterm, Final Exam, Q2). They let you draw proportional mixes of questions across different syllabus periods when generating a test.

  1. Click + New Exam Period.
  2. Select the parent Subject, then the parent Class.
  3. Enter an Exam Period Name and optional description.
  4. Click Save Exam Period.
ℹ️

Deleting a Subject or Class will not delete the questions tagged to them — their subject/class fields will appear as unlinked text. You can re-tag questions individually if needed.

Adding a Question

  1. Click + New Question in the header.
  2. Choose a Question Type from the dropdown.
  3. Type the Question Text. You may include any characters; special Unicode symbols (arrows, Greek letters, etc.) are automatically converted to safe ASCII equivalents in PDF exports.
  4. Optionally assign a Subject, Class, and Exam Period. The Class and Exam Period dropdowns are filtered by the selected Subject.
  5. Fill in the type-specific section (choices, pairs, table cells, etc.).
  6. Click Save Question.

Multiple Choice specifics

  • Set the number of choices (2–10) in the Number of Choices field.
  • Enter text for each choice, then click the Correct radio button next to the right answer.

Matching specifics

  • Set the number of pairs (2–10).
  • Enter left and right items for each pair. Right-column items are randomly shuffled when the test is generated so students cannot guess by position.

Table specifics

  • Enter the number of Columns and Rows, then click Build Table.
  • Type content into each cell.
  • Click the Q button on cells that should appear as blank answer fields on the student copy. Non-Q cells are hidden.
  • Click the B button on any cell to apply bold text formatting.

Adding an Image to a Question

Any question can optionally include one image, which will appear between the question text and the answer choices in the preview, on-screen cards, PDF, and DOCX exports.

How to attach an image

The Image (optional) field appears in the question form between the Question Text and the Subject/Class fields. You have three ways to add an image:

  • Paste a URL — Type or paste a web address (e.g., https://example.com/diagram.png) into the text field and click Preview (or press Enter). The image loads in a preview area below the field.
  • Browse for a local file — Click Browse… to open a file picker and choose any image file from your computer. The image is read directly into the app — no upload needed.
  • Drag and drop — Drag an image file from Explorer/Finder (or drag an image from a web browser) and drop it anywhere inside the dashed drop zone. The image loads immediately.
⚠️
Local file paths do not work when Quiz Keeper is hosted on a web server

If you are accessing Quiz Keeper through a web address (e.g., https://yourschool.example.com/quizkeeper), typing a path to a file on your own computer into the URL field — such as C:\Pictures\diagram.png or file:///Users/you/diagram.png — will not work. Browsers block this for security: a web-hosted page is not permitted to directly read files from your local drive via a typed path.

This limitation only affects the URL text field. The Browse button and drag-and-drop always work, regardless of whether the app is hosted on a web server or opened as a local file, because they use a secure browser mechanism that explicitly asks you to grant access to a specific file. Always use Browse or drag-and-drop when you want to use images from your own device.

Adjusting the display width

Once an image is previewed, an Image Display Width slider (10%–100%) appears. Drag it to set how wide the image should appear relative to the question column. This setting is saved with the question.

⚠️
Internet images and exports — what you need to know

When you use a web URL as your image source, the image shows fine in the browser preview. However, when you export to PDF or DOCX, the app needs to copy the actual image data into the file. This is only possible if the website that hosts the image gives explicit permission for it.

Think of it like this: your browser is allowed to display a photo from another site, but it may not be allowed to copy that photo into a document — the website owner controls that permission via something called a CORS policy. If the site does not grant this permission, the image will be silently skipped in the export (it will still show in the on-screen preview).

The safest approach for exports is to use the Browse or drag-and-drop method because local files are always fully embedded with no restrictions. Images saved this way are stored directly inside Quiz Keeper as data and will always export correctly.

ℹ️
Local file images and database backups

When you browse or drag-and-drop a local file, the image data is saved inside your Quiz Keeper database. This means it will also be included in your database backup — but be aware that backup file sizes will be larger for question banks with many images.

Editing & Deleting

Each question card in the Questions view has Edit and Delete buttons. Editing opens the same form pre-filled with the existing data. Deletion is immediate — there is no undo, so keep regular backups.

Searching & Filtering

Use the toolbar in the Questions view to narrow down the list:

  • Search box — matches against question text. Press Enter or click 🔍.
  • Subject / Class / Exam Period / Type dropdowns — any combination of filters can be active at once.
  • ✕ Clear — removes all active filters and shows the full question bank.

Generating a Test

Click Generate Test in the header to open the test configuration form.

  1. Enter a Test Title (required).
  2. Select the Subject. Only subjects with questions in the bank appear here.
  3. Optionally narrow to a specific Class and/or Exam Period.
  4. Use the Question Distribution sliders to set how many questions of each type to include.
  5. If no specific Exam Period was chosen, configure the Exam Period Mix (see below).
  6. Click Preview Test.

Question Distribution Sliders

After selecting a Subject (and optionally a Class and Exam Period), the sliders section shows one slider per question type that has available questions. Each slider shows the range 0 to the maximum available for that type.

  • The count you set is the number of questions that will be randomly drawn from the pool for that type.
  • Setting a slider to 0 excludes that type entirely.
  • The total question count updates in real time.

Exam Period Mix

This section appears only when no specific exam period is selected. It lets you specify what percentage of questions to draw from each exam period in the pool so a cumulative or mixed exam is proportionally balanced.

  • Each exam period has a percentage slider (0–100).
  • Sliders auto-balance as you adjust them so the total always reaches 100%.
  • The total indicator turns green at exactly 100% and red when over or under.
  • If a period doesn't have enough questions to satisfy its allocation, a Fallback dialog appears after clicking Preview Test, letting you redirect the shortfall to another period or omit the missing questions.
ℹ️

Questions not assigned to any exam period are labelled (No Exam Period) and can still be included in the mix.

Test Preview & Editor

After clicking Preview Test, a full live preview of the test is shown. It looks exactly as it will print. From here you can edit, save, randomise, and export.

Rich Edit Mode

Click 🔒 Locked to enable editing. The button changes to ✏️ Editing and a formatting toolbar appears.

  • Font size — 8pt to 32pt.
  • Text colour — colour picker applied to selected text.
  • Bold / Italic / Underline — standard rich text formatting.
  • Alignment — left, centre, or right for selected paragraphs.

Click inside any text in the preview, select the text you want to change, then apply formatting. Changes appear live. This implementation is currently not as feature rich as a full word processor so if you need more advanced editing for fonts, tabs, images etc. I suggest you edit the exported test file in your word processor of choice. The exception to this advice is removing questions from the test, if you remove or replace questions from the test preview screen, the test questions will be automatically renumbered. If you delete a question in a word processor, the question numbers are not a true numbered list so they will not automatically update.

⚠️

Rich-text formatting applied in the preview affects the printed/exported layout, but does not update the stored question text in the database. Use Edit Question for permanent question changes.

Adding / Removing Questions

Click 📝 Edit Questions in the preview footer to open the question editor panel.

  • Left panel — current questions in the test. Drag to reorder, or click the remove button to drop a question.
  • Right panel — the full question bank. Use the search box and filters to find questions, then click + Add to insert them into the test.
  • Click Save Changes to apply and return to the preview.

Refreshing Questions from the Bank

Saved tests store a snapshot of each question at the time the test was last saved. If you later edit a question in the bank — for example, adding an image to it — those changes will not automatically appear when you reopen the saved test.

Click 🔄 Refresh Questions in the preview footer to re-fetch every question in the current test from the question bank and update them to their latest version. The refreshed state is applied immediately to the preview.

⚠️
Refresh does not save automatically

Refreshing only updates the preview in memory. The saved test in Test History is not changed until you click 💾 Save Test. This means you can refresh and export an up-to-date version without overwriting the original snapshot — useful if you want to keep both.

A status message will appear after refreshing to tell you how many questions were updated. If a question has been deleted from the bank since the test was saved, it is kept as-is in the test.

Other Preview Actions

  • 🎲 Randomize Order — shuffles the question order in the current preview (does not affect the saved test).
  • 💾 Save Test — saves the test (with current question order and metadata) to Test History.
  • 📄 Export Test — opens the export format picker.

Exporting a Test

Click 📄 Export Test in the preview footer to open the export panel.

  • Tick Include Answer Key if you want the answers included inline with the questions and printed at the end of the document. The answer key and the non-answer key versions of the test should be exported separately.
  • Choose 📄 Export as PDF or 📝 Export as DOCX.

PDF Export

The PDF is generated in the browser using jsPDF and downloaded automatically. Formatting notes:

  • Font is Helvetica (standard PDF font). Special Unicode characters (arrows like →, Greek letters, superscripts, etc.) are automatically converted to readable ASCII equivalents.
  • Questions are kept intact across page breaks — a question will not be split between pages.
  • Short multiple choice options are laid out on one inline line; longer options are bulleted.
  • Matching questions shuffle the right-column items randomly on each export.
  • If an answer key is included, it is included inline with the questions and placed on a new page at the end.
  • Images — question images are embedded directly into the PDF file at an enlarged size for print clarity. See the image note about internet URLs if images are missing from your exported PDF.

The exported filename follows the format ClassName_TestTitle_YYYY-MM-DD.pdf (or Subject name if no Class is set).

DOCX Export

The Word document is generated using the docx.js library and downloaded automatically. It is compatible with Microsoft Word, LibreOffice Writer, and Google Docs.

  • Question blocks use Word's "keep together" paragraph property so they are not split across pages.
  • All header metadata, section instructions, and answer key options behave the same as in the PDF export.
  • Images — question images are embedded directly inside the .docx file (in the word/media/ folder of the document), so the file is fully self-contained. See the image note about internet URLs if images are missing from your exported DOCX.

The exported filename follows the format ClassName_TestTitle_YYYY-MM-DD.docx (or Subject name if no Class is set).

ℹ️
After export — save prompt

After exporting from the test preview window, a dialog will ask whether you want to save the current state of the test to history. Click OK to save (or update) the test, or Cancel to skip — the exported file is not affected either way.

Test History

Saved tests appear in the Test History tab as cards showing the title, question count, subject, class, and date saved.

  • Search — type in the search box and press Enter to filter by test title or subject.
  • Clear — removes the search filter and shows all saved tests.
  • Each Test card has action buttons to Load (reopens the preview so you can re-export or edit), Duplicate (creates a copy of the test), and Delete the test from history. Buttons for fast export to both formats are also included
ℹ️

Saved tests store a snapshot of the questions at the time they were saved. Editing or deleting a question in the bank will not change previously saved tests. Use the 🔄 Refresh Questions button in the preview footer to pull in the latest question data without permanently overwriting the saved snapshot.

Database Management

All database operations are accessed through ⚙️ Database in the header.

💾 Export Database

Downloads a complete snapshot of the database as a .json file containing all questions, subjects, classes, exam periods, and test history. This is your backup file.

🚨
Do this regularly.

Export your database after every session where you add or change data. Storing the file in at least two places (e.g., local drive and cloud storage) is the safest practice.

📂 Import Database

Merges a previously exported .json file into the current database.

⚠️
Watch for duplicates.

Import adds data on top of whatever is already in the database. If you want a clean restore, clear the database first, then import the backup.

🗑️ Clear Database

Permanently deletes everything — all questions, collections, and test history.

🚨
This cannot be undone.

Always export a backup before clearing. A confirmation dialog is shown before the operation proceeds, but once confirmed, the data is gone.

Tips & Best Practices

Organisation

  • Create all of your Subjects, Classes, and Exam Periods before you start entering questions. It is much faster to tag questions at entry time than to go back and re-tag them later.
  • Use Exam Periods as syllabus units (e.g., Chapter 1–3, Q1, Semester A). This unlocks the Exam Period Mix feature for balanced cumulative exams.

Generating Exams

  • For a cumulative test, leave the Exam Period selector blank and use the Exam Period Mix sliders to weight each unit proportionally.
  • Click 🎲 Randomize Order multiple times to generate several different question-order variants for different test versions.
  • Use Test Version in the header metadata to distinguish Version A, B, C, etc. when printing multiple variants.
  • You can load a previous test from Test History, modify it, refresh the questions to reflect the current state of the question bank, and export it for printing, and no changes will be made to that version unless you hit Save. This allows you to keep an archive of how that test was but still make changes for a one off use.
  • If you want to create a copy of a previous test for editing without possibly affecting the archive copy, Duplicate the previous test and work on the new copy. You'll be able to give it a new name and it's creation date will reflect when it was duplicated to avoid any confusion.

Exports

  • Export both a student copy (no answer key) and a teacher copy (with answer key) at the same time from the same preview.
  • DOCX files are easier to edit after export if you need last-minute changes. PDF is better for distribution to people that may not have a way or know how to open a Word/docx file.
  • If a PDF looks slightly different from the preview, it is because the browser render and the jsPDF render use different text engines. Use DOCX if more accurate layout is critical.

Data Safety

  • Never rely on a single backup. Keep copies in at least two separate locations.
  • After importing a backup, scroll through the Questions view to verify the data looks correct before continuing work.
  • Keep a dated archive of backups (e.g., one per week) so you can recover from accidental mass deletions or corruption.
  • If you use multiple computers, export on one machine and import on the other to transfer your question bank.
